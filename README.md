# object_tracking_drone

### 1. 서론

본 프로젝트는 Pixhawk 4(FC)와 NVIDIA Jetson(Companion Computer)을 활용하여 지능형 자율 비행 시스템을 구축한 연구 개발 프로젝트입니다. 

YOLOv8을 이용한 실시간 객체 인식과 Depth 카메라의 거리 데이터를 융합하여, 특정 목표물을 추적하고 일정한 간격을 유지하는 자율 비행 제어를 목표로 합니다.

### 2. 환경

•OS/Middleware: Ubuntu 20.04, ROS (MAVROS) 

•Hardware: Pixhawk 4, NVIDIA Jetson, Intel RealSense (T265, D435)

•Firmware/Software: PX4, QGroundControl (QGC), YOLOv8

### 3. 구현 내용

•하드웨어 통합 및 캘리브레이션

전력 시스템 구축: 배터리 전원이 분배 보드를 거쳐 ESC와 모터로 안정적으로 공급되도록 하드웨어 배선 및 납땜 작업을 직접 수행했습니다.

센서 보정: QGroundControl을 통해 지자기, 가속도계 캘리브레이션 및 ESC 캘리브레이션을 수행하여 모터 간 추력 균형을 확보했습니다.

•지능형 비행 제어 및 비전 시스템

객체 추적 알고리즘: YOLOv8을 통한 실시간 타겟 탐지와 D435 카메라의 Depth 데이터를 융합하여 목표물과의 거리를 산출하고 추적 제어 로직에 적용했습니다.

SLAM & Mapping: RealSense T265의 Pose 데이터와 D435의 Depth 데이터를 융합하여 RViz2 상에서 실시간 주변 환경 매핑을 구현했습니다.

PID 제어 최적화: 비행 테스트 중 발견된 Yaw 축 불안정 문제를 해결하기 위해 P, I, D 게인 값을 세밀하게 튜닝하여 기체의 거동을 안정화했습니다.

### 4. 트러블슈팅 및 개선

•소프트웨어 및 환경 구성 이슈

패키지 빌드 오류: MAVROS 설치 중 발생한 Git Clone 404 에러를 소스코드 수동 빌드 방식으로 해결하며 리눅스 패키지 통합 역량을 강화했습니다.

호환성 검토: 이기종 센서(어안 렌즈, MTF-01 등) 연동 시 발생하는 펌웨어 충돌 및 과열 문제를 해결하기 위해 기술 문서를 분석하고 수동 설치를 시도하며 시스템 안정성을 검토했습니다.

•기구학적 한계 극복

무게중심(CoG) 최적화: 소프트웨어 제어만으로 극복하기 어려운 기체 전방 쏠림 현상을 해결하기 위해 배터리 위치 조정 및 후면 카운터웨이트를 추가하여 물리적 무게중심을 밸런싱했습니다. 이를 통해 특정 모터의 부하를 분산시키고 비행 안정성을 근본적으로 개선했습니다.

### 5. 결과 및 보완점

객체 인식 후 뎁스 카메라를 통한 거리 계산까지는 정상적으로 작동했으나, 드론에 대한 배경지식이 많지 않아 또한 역학적 지식이 없어 거리 유지 코드가 정상적으로 작동하지 않아 큰 피해가 발생할 위기에 처했습니다. 

이를 보완하기 위해서는 드론의 비행 방식, 역학적 배경 지식, 드론 조종 숙달이 필요하다고 판단해 학습 후 재 진행 계획을 새웠습니다.
